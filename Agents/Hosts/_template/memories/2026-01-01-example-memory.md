# 2026-01-01 Example memory — delete me

The shape of a good memory note:

- **Dated filename**, short topic slug.
- **What changed / what was learned**, written for ANY future agent — no
  vendor pronouns, no "as I discussed earlier" (chat context dies; the repo
  doesn't).
- **Post-change evidence**: the command output or observed behavior that
  proves the claim. A memory about unverified work is forbidden — validate
  FIRST, then write.
- **Supersession discipline**: when a later session learns this note is
  stale, it edits the note ("superseded by ...", dated) rather than leaving
  the wrong version standing. Corrections show their history.
